## **フェーズ概要**

Phase 2の目的は、Phase 1で抽出した見込みリードに対して「短時間で作った試作サイト」を提示し、無理な売り込みではなく商談化のきっかけを作ることである。ここで求める成果は大量送信ではなく、個人でも回せる範囲で反応率と商談化率を安定させることにある。

このフェーズの基本方針は以下の3点に絞る。

- 試作は受注前の説得材料であり、本制作レベルまで作り込まない。
- 営業は1件ずつ質を担保しつつ、同じ業種テンプレートで再現性を上げる。
- 反応がない案件を追い過ぎず、Phase 3へ渡すべき温度感のある案件だけを前に進める。

Phase 2の最終成果物は、商談設定済みまたは具体的な相談が発生したリードの引き継ぎ情報一式である。

## **成功指標**

初月は検証、60日以降は安定運用を目標にする。以下はDay 60以降の定常目標。

| 指標 | 目標 | 判断ルール |
| --- | --- | --- |
| 月間試作生成件数 | 16〜20件 | 20件を超える前に返信率を確認する |
| 初回送付完了率 | 90%以上 | 試作生成から24時間以内に送れない案件は backlog に戻す |
| 試作URLアクセス率 | 30%以上 | 20件送って20%未満なら件名・DM文・サムネイルを修正する |
| 返信率 | 15%以上 | 20件送って8%未満ならチャネル配分を見直す |
| 商談化数 | 月2件以上 | 返信はあるが商談化しない場合は初回返信文を修正する |
| 受注数 | 月1件以上 | Day 90時点で未達なら対象業種か価格帯を再設計する |
| Phase 2起点の売上 | 月15万円以上 | 商談単価が低い場合はライト偏重をやめる |
| 1件あたりの作業時間 | 30分以内 | 40分を超える案件は個別対応を止めて保留にする |

## **前提条件**

Phase 2を回し始める前に、以下が揃っていることを前提とする。

- Phase 1でA判定、またはB判定でも連絡先が明確で改善余地が大きいリードがある。
- 店舗名、業種、住所、電話番号、口コミ抜粋、優先度、URLスラッグが揃っている。
- 試作テンプレートが最低1業種分あり、Vercelで即時公開できる。
- CRMが用意され、`送付日` `次回行動日` `ステータス` `試作URL` を必須入力にしている。
- 営業用のメール送信元ドメインにSPF/DKIMが設定済みである。
- Instagram運用を行う場合は、事業アカウントのプロフィール整備と最低限の投稿が完了している。
- 価格帯と標準プランが固定されており、初回返信で提示できる。
- 商談用の予約導線がある。例: Calendly、Google Meet、Zoom。

## **対象範囲と非目標**

### **対象範囲**

- リードの営業優先順位付け
- 試作サイトの生成、最低限のQA、公開
- 初回接触文の作成と送付
- 14日間の追客運用
- 反応内容の記録と商談化判断
- Phase 3へ渡すための要点整理

### **非目標**

- 受注前にフルカスタムデザイン案を作ること
- 受注前に詳細要件定義や詳細見積を作り込むこと
- 広告運用やSEO長期提案まで同時に売ること
- 反応のないリードに無制限で追客すること
- 契約交渉、法務確認、要件ヒアリングをこのフェーズで完了させること

## **実行ワークフロー**

| Step | 目的 | 実施内容 | 決定ルール | 時間上限 |
| --- | --- | --- | --- | --- |
| 1. キュー投入 | 今週動く案件を決める | A判定を上から確認し、5件を今週分に選ぶ | 初月は週5件、Day 60以降は週8件まで | 30分 |
| 2. 事前確認 | 試作を作る価値があるかを見る | Google Maps、Instagram、既存サイト、電話可否を確認 | チェーン店、本部サイト統一業態、閉業疑いは除外 | 10分/件 |
| 3. 試作生成 | URL付き提案を作る | テンプレートに店舗データを差し込み、コピーを生成してビルドする | 2回失敗したら手作業で追わず保留にする | 10〜15分/件 |
| 4. QAと公開 | 信頼を落とさない最低品質を担保する | 店舗名、電話、住所、営業時間、CTA、モバイル表示を確認する | 事実確認できない文言は削る。推測で補わない | 5分/件 |
| 5. 初回送付 | 接触を開始する | 最適チャネルを選び、試作URL付きで送る | 試作公開から24時間以内に送れない場合は翌週へ回す | 5分/件 |
| 6. 行動観測 | 温度感を測る | URLアクセス、返信、プロフィール閲覧、電話反応を記録する | アクセスありは優先追客、無反応は定型追客へ | 5分/件 |
| 7. 追客 | 商談化する | Day 3、Day 7、Day 14で追客する | 14日で反応がなければ60日後再アプローチへ移す | 5分/件 |
| 8. 選別と引き継ぎ | Phase 3対象を明確にする | 興味度、予算感、課題感、時期感を整理する | 商談設定または具体相談が発生した案件のみ引き継ぐ | 10分/件 |

## **週次オペレーション**

一人運営では「毎日少しずつ」より「曜日ごとに役割を固定する」方が抜け漏れが少ない。標準運用は週8〜12時間を目安にする。

| 曜日 | 主作業 | 具体タスク | 目安時間 |
| --- | --- | --- | --- |
| 月曜 | 仕込み | 今週分リード選定、事前確認、業種別キュー整理 | 1.5時間 |
| 火曜 | 試作生成 | 2〜3件の試作作成、公開、CRM登録 | 2時間 |
| 水曜 | 初回送付 | メール/DM送付、送付ログ記録、未送付案件処理 | 1.5時間 |
| 木曜 | 追客 | Day 1確認、Day 3追客、電話枠運用 | 1.5〜2時間 |
| 金曜 | 商談と改善 | 商談実施、メモ整理、KPI確認、翌週仮説設定 | 1.5〜2時間 |

### **週次の運用ルール**

- 新規試作は週前半に寄せる。金曜に新規送付を増やさない。
- 電話は木曜または金曜の14:00〜17:00にまとめる。
- 同時に追う新規案件は10件以内に抑える。
- `次回行動日` が未設定のリードを残したまま週を終えない。

## **チャネル戦略**

チャネル選択は「相手が見やすいか」と「こちらが継続運用できるか」で決める。

### **優先順位**

1. Instagram DM
2. メール
3. 電話
4. その他の問い合わせフォーム

### **選択ルール**

- Instagramアカウントがあり、直近30日以内に投稿があるならDMを第一候補にする。
- Instagramはあるが更新停止中で、メールが取得できるならメールを優先する。
- メールがなく電話のみある場合、初回電話の目的は売り込みではなく「試作URL送付先の確認」に限定する。
- 飲食、美容、整体など来店型ビジネスで電話番号がある場合、Day 7時点で無反応なら電話を使う。
- 同じ日に2チャネル以上で新規接触しない。圧迫感を避ける。
- 返信があったチャネルを主チャネルとし、以後の連絡は相手の負担が少ない方法に寄せる。

### **送付上限**

- Instagram DM: 1日20件まで
- メール: 1日30件まで
- 新規電話: 1日8件まで
- 追客電話: 1日5件まで

### **チャネル別の判断メモ**

| チャネル | 向いている相手 | 強み | 注意点 |
| --- | --- | --- | --- |
| Instagram DM | カフェ、美容、飲食、写真映え業種 | 開封されやすく、試作URLとの相性が良い | アカウント品質が低いとスパム扱いされやすい |
| メール | 法人色の強い店舗、BtoB寄り業種 | 説明と価格感を丁寧に伝えやすい | 件名と送信ドメイン品質で差が出る |
| 電話 | 地域密着店、営業時間が読みやすい業種 | 温度感を即確認できる | 時間帯を間違えると印象を落とす |

## **試作サイト生成フロー**

Phase 2の試作は「受注前提のミニマム提案物」であり、毎回ゼロから設計しない。

### **基本フロー**

1. Phase 1のリードデータを読み込む。
2. 業種別テンプレートを選ぶ。
3. 店舗情報、口コミ、営業時間、写真を差し込む。
4. AIでヒーローコピー、紹介文、レビュー見出し、CTA文を生成する。
5. ビルドして公開URLを取得する。
6. モバイル表示と事実誤認だけ確認する。
7. CRMに試作URLと送付準備状況を記録する。

### **生成時の運用ルール**

- 業種テンプレートは最大3種類までに絞る。増やし過ぎない。
- 1件の試作で使う口コミは最大3件までにする。
- 誇大表現、根拠のない最上級表現、事実未確認の強みは書かない。
- 写真が不足している場合は無理にギャラリーを増やさず、ヒーロー1枚に絞る。
- 既存サイトがある店舗には「ゼロから新規制作」ではなく「スマホ最適化・情報整理」寄りの見せ方にする。
- 1件の試作で行う手直しは公開前1回までに制限する。
- 公開URLは60日で整理対象にする。

### **情報不足時のフォールバック**

| 不足情報 | 対応ルール |
| --- | --- |
| 写真がない | 業種別の既定ビジュアルに切り替え、実写のように見せない |
| 営業時間が不明 | セクションに載せず、問い合わせ誘導に寄せる |
| 口コミが1〜2件しかない | 使える件数だけ使い、件数を盛らない |
| Instagramがない | CTAは電話または問い合わせ導線に統一する |
| 電話番号が使えない | 送付対象から外すか、メール取得後まで保留にする |

### **公開前の最低QA**

- 店舗名、住所、電話番号が一致している
- レビュー引用が元情報と矛盾していない
- モバイルファーストで閲覧できる
- CTAが1つに絞られている
- 「試作サイト」であることを説明文か営業文で補足できる

## **CRM・追客ルール**

Phase 2で抜けやすいのは、試作を作ることではなく、その後の追客と判定である。CRMでは必ず「前進」か「保留」かを明確にする。

### **必須ステータス**

- 試作生成待ち
- 試作公開済み
- 送付済み
- アクセスあり
- 返信あり
- 商談中
- 受注
- 不成立
- 60日後再アプローチ

### **必須入力項目**

- 店舗名
- lead_id
- 業種
- 優先度
- 試作URL
- 初回送付日
- 最終接触日
- 次回行動日
- 送付チャネル
- 反応メモ

### **追客ルール**

| タイミング | 条件 | 実施内容 | ステータス |
| --- | --- | --- | --- |
| Day 0 | 初回送付 | URL付きで送付し、次回行動日をDay 1に設定 | 送付済み |
| Day 1 | URLアクセスあり | 「ご確認いただけましたか」の短い確認文を準備 | アクセスあり |
| Day 3 | URLアクセスあり、返信なし | 確認依頼の追客を行う | アクセスあり |
| Day 3 | URLアクセスなし | 文面を変えずに再送する | 送付済み |
| Day 7 | 無反応 | 別チャネルへ切り替える | 送付済み |
| Day 14 | 無反応 | ソフトクローズし、60日後再アプローチ日を設定 | 60日後再アプローチ |

### **返信内容ごとの判断ルール**

- 「興味あります」「相談したい」: 当日中に商談候補日時を返し、`商談中` に進める。
- 「料金を知りたい」: プランレンジだけ返し、値引きはしない。2営業日以内のフォローを入れる。
- 「今は考えていない」: 明確な断りなら `不成立`、時期未定なら `60日後再アプローチ` にする。
- 「社内確認します」: 返答予定日を必ず取る。取れない場合は3営業日後に確認する。
- URLアクセスのみで返信なし: 温度感ありとみなすが、追客は最大2回までに留める。

### **守るべき運用原則**

- 1リードにつき初回送付を含め最大4接触までにする。
- すべてのリードに `次回行動日` を入れる。空欄は放置と同義とみなす。
- 価格の詳細見積は商談前に出さず、レンジ提示で止める。
- 追客メモは感想ではなく事実を書く。例: 「5/24 DM既読、返信なし」。

## **コストガードレール**

Phase 2は売上化前の投資フェーズなので、金額よりも「小さく検証してから増やす」ことを優先する。

| 項目 | ガードレール | 超過時の対応 |
| --- | --- | --- |
| AI生成コスト | 試作1件あたり10円以内 | リトライ回数を減らし、プロンプトを固定化する |
| 月間試作数 | Day 30までは10件、Day 60以降は20件まで | 返信率が改善するまで増やさない |
| 人的工数 | 1件30分以内 | 個別カスタムを止めてテンプレートへ戻す |
| Vercel公開数 | 60日超の試作は整理対象 | 商談中以外は月末に棚卸しする |
| 月間ツール費 | 受注前は5,000円以内を目安 | 新規ツール導入を止める |
| DM/メール送付量 | 既定上限を超えない | アカウント健全性を優先する |

### **停止条件**

- 20件送ってアクセス率20%未満なら、件名かDM文を直すまで新規送付を止める。
- 返信率8%未満が2週続いたら、対象業種を1つに絞って再検証する。
- 商談化しても予算不一致が続くなら、ライトプラン訴求をやめて標準単価を引き上げる。

## **リスク登録簿**

| リスク | 兆候 | 影響 | 対応策 |
| --- | --- | --- | --- |
| 試作の品質不足 | URLは見られるが返信がない | 信頼を落とす | ファーストビューとコピー品質を最優先で改善する |
| 事実誤認の掲載 | 店舗情報に誤りがある | 即失注につながる | QAで事実確認できない文言は削除する |
| DM/メールのスパム判定 | 既読率や到達率が急落する | チャネルが使えなくなる | 送付上限を守り、送信元品質を維持する |
| 追客漏れ | 次回行動日が空欄の案件が増える | 商談機会を逃す | 金曜にCRM棚卸しを必須化する |
| 無駄な作り込み | 1件40分以上かかる | 継続運用できない | 受注前の修正上限を1回に固定する |
| 価格だけ聞かれて終わる | 返信はあるが商談にならない | 低単価化する | レンジ提示に留め、課題と時期の確認を返す |
| Phase 3への引き継ぎ不足 | 商談後の再質問が多い | 進行が遅れる | 商談前後の要点をテンプレートでまとめる |

## **Phase 3への引き継ぎ**

Phase 2の役割は、受注を確定することではなく「契約・要件整理に進めるべき案件を選ぶ」ことである。

### **引き継ぎ条件**

以下のいずれかを満たした時点でPhase 3へ渡す。

- 商談日時が確定した
- 具体的な制作相談が来た
- 予算レンジに納得感があり、導入時期の話に進んだ
- 予約機能、CMS、採用ページなど要件の話が出始めた

### **引き継ぎしない条件**

- 単なる挨拶返信のみ
- 価格確認だけで温度感が低い
- 店舗側の意思決定者が不明
- 時期感がなく、検討フェーズにも入っていない

### **引き継ぎ内容**

- 店舗基本情報
- 試作URL
- 初回送付日と使用チャネル
- アクセス有無と返信履歴
- 相手が反応したポイント
- 推薦プランと価格レンジ
- 想定課題
- 商談までに確認すべき未解決事項

### **引き継ぎSLA**

- 商談設定後、当日中にCRM更新を完了する
- 商談実施後、24時間以内にPhase 3向けメモを確定する

## **30/60/90日展開**

### **Day 0-30**

目的は「売れる型」を見つけること。対象業種は1つに絞る。

- 週5件までの試作で十分
- チャネルはDMとメール中心
- 試作テンプレートは1種類に固定
- A/Bテストは件名とDM冒頭だけに限定
- 目標はアクセス率20%以上、返信率8%以上

### **Day 31-60**

目的は「再現性」を作ること。勝ち筋の業種に絞って件数を少し増やす。

- 週6〜8件へ拡大
- Day 7の電話追客を導入
- 件名、DM文、CTAを継続的に改善
- 商談後の引き継ぎテンプレートを固定化
- 目標はアクセス率25%以上、返信率12%以上、商談月2件

### **Day 61-90**

目的は「安定運用」にすること。案件化率と単価の両方を整える。

- 月16〜20件の試作・送付を安定化
- 業種テンプレートを最大3種類まで増やす
- 低単価案件を減らし、スタンダード以上を主軸にする
- 60日再アプローチの再開運用を始める
- 目標は受注月1件以上、Phase 2起点売上15万円以上

## **Done Checklist**

1件のリードに対してPhase 2が完了したと言えるのは、以下のどちらかに到達したときである。

### **Phase 3へ進む場合**

- [ ]  試作URLが作成・記録されている
- [ ]  初回送付と追客履歴がCRMに残っている
- [ ]  相手の反応内容が記録されている
- [ ]  商談日時または具体相談が発生している
- [ ]  推薦プランと価格レンジが整理されている
- [ ]  未解決事項を含めてPhase 3へ引き継げる状態である

### **クローズする場合**

- [ ]  14日間の追客が完了している
- [ ]  不成立または60日後再アプローチの判断が済んでいる
- [ ]  次回行動日が空欄で残っていない
